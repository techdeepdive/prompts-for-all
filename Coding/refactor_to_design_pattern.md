# Design Pattern Refactoring

## Explanation
Identifies if a piece of messy code could benefit from a gang-of-four design pattern.

## Detailed Prompt
Copy and paste the prompt below:

```text
Analyze the following code snippet: [Insert Code]. It works, but it feels tightly coupled and hard to maintain. Suggest the best Software Design Pattern (e.g., Strategy, Factory, Observer) that fits this use case. Rewrite the code applying this pattern, and explain why this specific pattern makes the code more scalable.
```
